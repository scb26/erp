Dependency Order

Create these tables in this order:

  1. roles
  2. users
  3. companies
  4. customers
  5. products
  6. invoices
  7. invoice_items
  8. payments

This order works because each table is created only after the tables it references already exist.

Practical Development Order

  1. companies
  2. customers
  3. products
  4. invoices
  5. invoice_items
  6. payments
  7. users
  8. roles

This is the more practical order for current Unidex development because company, customer, product, and billing are the first business modules in use.
